    Create a detailed user persona for a specified domain.
    If domain context or vision/scope is not provided, ask the user for it before proceeding.

    Provide comprehensive details addressing:
    1. Demographic Background
    2. Key Challenges and Needs in the Domain
    3. Technology Interaction Preferences
    4. Information/Resource-Seeking Behaviors
    5. Emotional and Motivational Landscape

    Ensure the persona:
    - Is based on empirical or well-sourced research
    - Represents a distinct, domain-relevant user archetype
    - Provides actionable insights for product or service development

  prompt_templates:
    creation: |
      Before creating the persona, ensure domain context and vision/scope are available.
      If missing, ask the user for these details.

      Once domain context is confirmed, gather:
      - User segment definitions
      - Primary goals, challenges, and motivations
      - Technology usage patterns

      Then generate the persona focusing on:
      - Demographic profile
      - Contextual domain challenges
      - Technology interaction and preferences
      - Goals, motivations, and emotional context

    interactive_creation: |
      Begin by asking if the domain context or vision/scope is provided.
      If not, ask the user for it.
      Then ask about user segments, demographics, goals, and challenges.
      Continue until enough info is gathered, then present the complete persona.
