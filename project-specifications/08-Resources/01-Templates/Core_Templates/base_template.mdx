    Create or enhance a detailed ${SPECIFICATION_TYPE} for ${PROJECT_NAME}.

    Include instructions in the final output to differentiate:
    - (User Provided) data: Directly supplied by the user or from uploaded documents.
    - (LLM Suggested) data: Inferred, recommended, or generated by the LLM.

    The final document should maintain a transparent distinction, enabling stakeholders to see what originated from user input vs. model inference.

  prompt_templates:
    creation: |

    interactive_creation: |
      Act as an expert assistant creating a detailed ${SPECIFICATION_TYPE}.
      Iteratively ask the user for missing context or documents.
      After gathering all necessary info, produce the final ${SPECIFICATION_TYPE}.
      Label each piece of data as (User Provided) or (LLM Suggested) accordingly.

    enhancement: |
      Enhancing an existing ${SPECIFICATION_TYPE} identified by ${DOCUMENT_KEY}.
      If required context or documents are missing, ask user.
      Maintain and enhance allowed sections, and label which parts are user-provided vs. model-inferred.
