## AI-Assisted Persona Development
ai_assistance:
  document_type: "user_persona"
  parent_template: "adaptive_persona_framework"

ai_prompting_instructions:  

  prompt_refinement_checklist:
    - verify_data_sources
    - check_bias_potential
    - ensure_privacy_protection
    - validate_representativeness


  base_prompt_structure: |
    Create  a detailed user persona for a [USER_TYPE] in the healthcare context, or enhance an existing persona if identified with a [DOCUMENT_KEY].
    
    Provide comprehensive details addressing:
    1. Demographic Background
    2. Health Journey and Challenges
    3. Technology Interaction Preferences
    4. Information Seeking Behaviors
    5. Emotional and Psychological Landscape
    
    Ensure the persona:
    - Is based on empirical research
    - Represents a distinct user archetype
    - Provides actionable insights for product development

  prompt_templates:
    creation: |
      Generate a comprehensive user persona for ${PROJECT_NAME}
      Focus Area: ${USER_SEGMENT}
      Key Considerations:
      - Contextual Background: ${USER_CONTEXT}
      - Primary Objectives: ${USER_PRIMARY_GOALS}
      - Critical Challenges: ${USER_KEY_CHALLENGES}


    interactive_creation: |
      You will act as an assistant to help create a detailed user persona. User can provide context about the whore project or the persona being created.
      Begin by asking me questions to gather necessary information.
      Use the following guidelines:
      - Ask one question at a time.
      - Use previous answers to inform subsequent questions.
      - Continue until you have enough information to create the persona.
      - At the end, present the complete user persona using the template structure. 

    enhancement: |
      Act as an expert user research assistant. Your task is to **enhance** the existing persona identified as ${DOCUMENT_KEY}.

      **Specific Instructions**:
      
      - **Do not modify** any section without asking to the user and this agree with the modifications
      - **Maintain** all existing details in all the sections if the user does not provide any new information or does not agree to modify the existing details.
      - **Identify** gaps or areas that can be enriched in the allowed sections.
      - **Incorporate** new insights or provided data, **without altering** the sections that must remain the same.
      - **Ensure** consistency and accuracy throughout the persona.
      - **Justify** any additions or changes based on context or new data provided.
      - **Validate** against latest research
           

      **Process**:
      1. Review the existing persona.
      2. **Skip** any modifications in the specified sections that must remain the same.
      3. Identify areas for improvement in other sections and gather new insights and confirm with the user.
      4. Add additional details where necessary, **without changing** existing details in protected sections.
      5. Present the updated persona following the template structure.

      **Note**: The goal is to **enrich** the persona by improving the allowed sections, **without recreating** it or modifying details in sections that must remain the same.
